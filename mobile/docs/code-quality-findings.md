# Code Quality Findings

> Generated by code-review pipeline Phase 1-2

## 1. Dead Code / No-Op Code (BUG)

### Double pack() call in integration tests
- **Location:** `test/integration_test.dart:266-268` (CursorState roundtrip)
- **Location:** `test/integration_test.dart:317-318` (ScreenSnapshot roundtrip)
- **Problem:** `pack()` is called twice — first result is discarded, second is used. This wastes buffer space and cycles. Correct pattern is `builder.finish(cursorT.pack(builder))` (single pack, result used immediately).
- **Fix:** Change from:
  ```dart
  cursorT.pack(builder);
  builder.finish(cursorT.pack(builder));
  ```
  To:
  ```dart
  builder.finish(cursorT.pack(builder));
  ```

### Dead test - never calls the conversion function
- **Location:** `test/integration_test.dart:263-276` (CursorState roundtrip)
- **Problem:** The test verifies raw FlatBuffers serialization but never actually tests `cursorStateFromFlatBuffer()` from the ScreenConverter. It duplicates testing already done elsewhere.
- **Fix:** Either remove this test (redundant with ScreenConverter group), or rename it to explicitly test only raw FB serialization.

## 2. Duplication

### FlatBuffers builder pattern repeated 6 times
- **Location:** `lib/services/grpc_client.dart` (lines 85-88, 126-129, 152-155, 172-175, 192-195, 206-209)
- **Problem:** Every gRPC method identically constructs `fb.Builder`, packs a message, finishes it, and creates a request. This pattern should be extracted to a helper.
- **Fix:** Add a private helper:
  ```dart
  List<int> _buildRequest<T extends fb.Packable>(T object) {
    final builder = fb.Builder(deduplicateTables: false);
    final offset = T.pack(builder, object);
    builder.finish(offset);
    return builder.buffer;
  }
  ```
  Then simplify each method to use `get_snapshot(_buildRequest(GetSnapshotRequestT(sessionName: session)))`.

### _drawDashedLine and _drawDottedLine nearly identical
- **Location:** `lib/widgets/terminal_grid.dart:289-333`
- **Problem:** Two functions with identical logic except `dashLength = 4.0` vs `dotLength = 2.0`.
- **Fix:** Unify into `_drawSegmentedLine(segmentLength, gapLength, ...)` with callers passing appropriate values.

### Cell construction with default color duplicated
- **Location:** `lib/utils/screen_converter.dart:83-91` vs `lib/models/screen_buffer.dart:52-65`
- **Problem:** Both files independently construct `const Cell(ch: 0, fg: CellColor(...), bg: CellColor(...), flags: 0)`.
- **Fix:** Add a static factory `Cell.empty()` to the Cell class, or define a module-level `emptyCell` constant.

## 3. Silent Failures

### applyScreenUpdate silently skips out-of-bounds
- **Location:** `lib/utils/screen_converter.dart:78, 98`
- **Problem:** Out-of-bounds rows/cols are silently skipped with `continue`. If relay sends bad deltas, the error is invisible.
- **Fix:** Add debug logging:
  ```dart
  if (row < 0 || row >= screen.rows) {
    debugPrint('applyScreenUpdate: row $row out of bounds (0-${screen.rows})');
    continue;
  }
  ```

## 5. Unsafe Pattern

### FocusNode created inline but never disposed
- **Location:** `lib/screens/terminal_screen.dart:237`
- **Problem:** `FocusNode()..requestFocus()` creates a FocusNode that is never stored or disposed, causing a memory leak.
- **Fix:** Store as instance variable and dispose in `dispose()`:
  ```dart
  final _focusNode = FocusNode();
  // ...
  @override
  void dispose() {
    _focusNode.dispose();
    super.dispose();
  }
  ```
